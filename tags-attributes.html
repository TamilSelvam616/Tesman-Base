HTML Tags & Attributes

1. Document Foundation

  • <!DOCTYPE html>: This tells the browser that your HTML file is an HTML5 document, like a signboard. It must be the first line of the file. It’s not case-sensitive, so you can use capital or small letters. It ensures the browser works correctly.
    Example:
    <!DOCTYPE html>
  • <html>: This is like a big box that holds all HTML content. Everything goes inside it. Attributes like lang="en" set the language (helps with accessibility) and dir="rtl" makes text right-to-left, like for Arabic.
    Example:
    <html lang="en"></html>

2. Metadata Masters

  • <head>: This gives instructions to the browser and search engines but isn’t visible on the page. It contains the title, meta tags, CSS links, etc.
    Example:
    <head><title>My Page</title></head>
  • <title>: Shows the page’s name in the browser tab and Google search. It should be short and clear.
    Example:
    <title>Tesman Learning</title>
  • <meta>: Provides extra details. charset="UTF-8" supports all language characters. name="viewport" makes the page fit properly on mobile. name="description" appears as a snippet in Google search.
    Example:
    <meta charset="UTF-8">
<meta name="viewport" content="width=device-width">
<meta name="description" content="Learn coding">
  • <link>: Connects external files to the page, like a CSS file for styling or a favicon for the browser tab icon.
    Example:
    <link rel="stylesheet" href="styles.css">
<link rel="icon" href="favicon.ico">

3. Content Organizers

  • <header>: The top part of the page, like a logo or menu. <footer>: The bottom part, like contact or copyright info.
    Example:
    <header><h1>Tesman</h1></header>
<footer><p>Contact Us</p></footer>
  • <nav>: Holds navigation links, like a menu bar. Screen readers prioritize it.
    Example:
    <nav><a href="home.html">Home</a></nav>
  • <main>: The main content of the page; only one per page.
    Example:
    <main><article>Content</article></main>
  • <article>: Independent content, like a blog post or news article, that can stand alone.
    Example:
    <article><h2>Blog Post</h2></article>
  • <section>: Groups related content, like a book chapter with a heading.
    Example:
    <section><h3>Chapter 1</h3></section>
  • <div>: A general box for grouping content, often used for styling.
    Example:
    <div><p>Text</p></div>
  • <span>: Used to style small inline text, like highlighting a single word.
    Example:
    <span>Highlighted</span>

4. Text Hierarchy

  • <h1>-<h6>: Headings, from <h1> (biggest) to <h6> (smallest). One <h1> per page is good for SEO. Screen readers use them for navigation.
    Example:
    <h1>Main Title</h1>
<h2>Subheading</h2>
  • <p>: Paragraphs for text blocks, automatically adds space above and below.
    Example:
    <p>This is a paragraph.</p>
  • <strong>: Makes important text bold.
    <em>: Makes emphasized text italic.
    <mark>: Highlights text (usually yellow).
    Example:
    <strong>Important</strong>
<em>Emphasized</em>
<mark>Highlighted</mark>

5. Media Elements

  • <img>: Displays images on the page. src specifies the image path, alt provides an accessibility description, loading="lazy" speeds up page load.
    Example:
    <img src="photo.jpg" alt="Scenery" loading="lazy">
  • <video> and <audio>: Plays video or audio. controls adds play/pause buttons, autoplay plays automatically (browsers may block), loop repeats the media.
    Example:
    <video src="vid.mp4" controls loop></video>
<audio src="song.mp3" controls></audio>

6. Navigation & Links

  • <a>: Creates links. href points to a URL or section, target="_blank" opens in a new tab (use rel="noopener" for security), download triggers a file download.
    Example:
    <a href="https://tesman.com" target="_blank" rel="noopener">Visit</a>
<a href="file.pdf" download>Download</a>

7. Data Organization

 
  • <table>: Creates a table. <tr>: Row. <th>: Header cell (scope="col/row" for accessibility). <td>: Data cell (colspan/rowspan merges cells).
    Example:
    <table>
  <tr><th scope="col">Name</th></tr>
  <tr><td>John</td></tr>
</table>
  • <ul> and <ol>: Bullet list (unordered) or numbered list (ordered). <ol> attributes: reversed, start. <li>: List item.
    Example:
    <ul><li>Item 1</li></ul>
<ol start="2"><li>Step</li></ol>

8. User Input (Forms)

  • <form>: Collects user input, like login or survey forms. action specifies where data is sent, method="GET/POST" defines how data is sent, autocomplete controls form auto-fill.
    Example:
    <form action="/submit" method="POST">
  <input type="text">
</form>
  • <input>: Creates input fields. Types: text, email, password, checkbox, radio, file (accept=".pdf" restricts file types), date, color, range.
    Example:
    <input type="email" required>
<input type="file" accept=".pdf">
  • Special Attributes: required makes a field mandatory, pattern="[A-Za-z]" validates input, placeholder shows hint text.
    Example:
    <input type="text" placeholder="Enter name" required>

9. Interactive Elements

  • <details> and <summary>: Creates a clickable content box that expands, works without JavaScript.
    Example:
    <details>
  <summary>Click me</summary>
  <p>Hidden content</p>
</details>
  • <dialog>: Creates a popup or modal window. Use the open attribute or JavaScript (.showModal()) to display.
    Example:
    <dialog open>Popup content</dialog>

10. Global Attributes

  • Identification: id is a unique identifier, one per page. class creates reusable style groups.
    Example:
    <div id="unique" class="style">Content</div>
  • Accessibility: aria-label provides descriptions for screen readers, tabindex controls keyboard navigation.
    Example:
    <button aria-label="Close" tabindex="0">X</button>
  • Custom Data: data-* stores custom info for JavaScript, like data-user-id.
    Example:
    <div data-user-id="123">User</div>

Key Concepts to Remember

  • Semantics Over Style: Choose tags based on their meaning, not appearance.
    Example:
    <header><h1>Title</h1></header>
  • Attribute Order: Order doesn’t technically matter, but alphabetical order is common.
    Example:
    <img alt="Photo" src="image.jpg">
  • Progressive Enhancement: Ensure the page works without CSS or JavaScript.
    Example:
    <form action="/submit"><input type="text"></form>
  • Accessibility: Screen readers rely on correct tags and attributes.
    Example:
    <button aria-label="Submit">Send</button>
