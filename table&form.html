Master HTML Tables & Forms

Learn how to create structured data layouts and interactive user inputs with HTML tables and forms. Essential skills for modern web development.

HTML Tables and Forms

1. HTML Tables

Tables organize data in rows and columns, ideal for schedules, product lists, or comparison charts. Key tags include <table> for the table container, <tr> for rows, <th> for headers (with scope for accessibility), and <td> for data cells. Attributes like colspan and rowspan merge cells across columns or rows.

<table>
    <thead>
        <tr>
            <th scope="col">Course</th>
            <th scope="col">Instructor</th>
            <th scope="col">Duration</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td colspan="2">HTML & CSS</td>
            <td>4 weeks</td>
        </tr>
        <tr>
            <td>JavaScript</td>
            <td>Jane Doe</td>
            <td>6 weeks</td>
        </tr>
    </tbody>
</table>

Output: Table

Visual Description: The table has a dark background with subtle borders. Headers have a blue accent background, and rows have a hover effect for better interactivity. The "HTML & CSS" cell spans two columns, creating a wider cell that aligns with the single "4 weeks" cell in the Duration column.

2. HTML Forms

Forms collect user input for tasks like registration or feedback. The <form> tag defines the form, with attributes like action (where data is sent) and method (GET or POST). Input elements like <input>, <select>, and <textarea> handle different data types, with attributes like required and pattern for validation.

<form action="/submit" method="POST">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" placeholder="Enter your name" required>
    <label for="course">Course:</label>
    <select id="course" name="course">
        <option value="">Select a course</option>
        <option value="html">HTML & CSS</option>
        <option value="js">JavaScript</option>
    </select>
    <label for="feedback">Feedback:</label>
    <textarea id="feedback" name="feedback" rows="4" placeholder="Your feedback"></textarea>
    <button type="submit">Submit</button>
</form>

Output: Form

The form below allows users to enter their name, select a course, and provide feedback.

Visual Description: The form is vertically stacked with bold white labels and dark input fields with subtle borders. The text input, dropdown, and textarea have a consistent design with focus states that highlight with a blue glow. The submit button uses the primary gradient and has a hover effect.

3. Combining Tables and Forms

Combining tables and forms creates interactive data displays, like editable tables for managing records. Each table row can contain a form with inputs for updating data, often used in dashboards or admin panels.

<table>
    <thead>
        <tr>
            <th scope="col">Student</th>
            <th scope="col">Course</th>
            <th scope="col">Grade</th>
            <th scope="col">Actions</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>John Doe</td>
            <td>HTML & CSS</td>
            <td>
                <form action="/update-grade" method="POST">
                    <input type="hidden" name="student" value="John Doe">
                    <input type="text" name="grade" value="A" pattern="[A-F]" required>
                    <button type="submit">Update</button>
                </form>
            </td>
            <td>
                <form action="/delete-student" method="POST">
                    <input type="hidden" name="student" value="John Doe">
                    <button type="submit">Delete</button>
                </form>
            </td>
        </tr>
    </tbody>
</table>

Output: Table + Form

The table below shows a student roster with forms in each row to update grades or delete students.

Student Course Grade Actions
John Doe HTML & CSS
Jane Smith JavaScript

Visual Description: The table has a dark background with subtle borders and hover effects. Each row contains two forms: one for updating grades (with a text input restricted to A-F grades) and one for deleting the student. The update buttons use the primary gradient, while delete buttons have a red accent. The layout is compact with forms fitting neatly within table cells.

Key Takeaways

  • Tables are essential for displaying structured data in a readable format
  • Forms enable user interaction and data collection on websites
  • Combining tables and forms creates powerful data management interfaces
  • Always use proper accessibility attributes like scope in tables and label elements in forms
  • Responsive design ensures tables and forms work well on all device sizes
